Portal Audit Report

Marc Watters - Construction Business Blueprint | 19 March 2026

5 Critical 3 Bugs 5 Incomplete 7 Nice to Have

Go-Live Blockers

Fix items 1, 2, 4, 5 before any real client touches the portal. Item 3 (Stripe) is needed when ready to take payments. Everything else can ship after launch.

!
Critical
5 issues
#1
Bulk Assign API has no auth check
/api/admin/bulk-assign uses the Supabase service role key but has zero authentication. Anyone who knows the endpoint can POST to it and assign training modules to any client. Needs requireAdmin() added.
#2
Demo user fallback leaks client data
7 portal API routes (dashboard, plan, me, ai, avatar, settings, training) fall back to the first client in the database if no user is authenticated. This means an unauthenticated request returns real client data - business plans, check-in history, training progress. These demoUser fallbacks must be removed before go-live.
#3
No Stripe keys configured
.env.local has no STRIPE_SECRET_KEY or STRIPE_WEBHOOK_SECRET. The payment intent endpoint returns demo_mode. The Stripe webhook rejects everything. Automated onboarding (Stripe checkout -> user creation -> module assignment) won't work. Not a blocker if Marc is manually adding clients for now.
#4
GHL form ID is a placeholder
The /apply page has YOUR_FORM_ID hardcoded in the GHL iframe src. The application form is completely broken. Need Marc's actual GHL form ID to replace it.
#5
Resend domain not verified
The Resend API key is configured but the sending domain (marc@marcwatters.com) isn't verified. All emails silently fail: welcome emails, check-in reply notifications, and client invite emails. Domain verification needs done in the Resend dashboard.
⚠
Bugs
3 issues
#6
Payment routes have no auth
/api/create-payment-intent and /api/schedule-payment are public endpoints. While they won't work without Stripe keys, once Stripe is configured anyone could create payment intents. Should validate the request or add rate limiting.
#7
Onboarding video is a placeholder
The /onboarding page shows "Onboarding video - to be added by Marc" with a fake play button. Marc needs to record a welcome video and provide the YouTube/Vimeo URL to embed.
#8
Invite client password setup link needs testing
The invite flow generates a Supabase recovery link and constructs a callback URL with token_hash. The auth callback route needs to be verified that it handles this token type and redirects correctly to set a password.
⚙
Incomplete
5 items
#9
Weekly summary email to Marc
Noted as Phase 4 scope. Marc has no automated weekly digest of client activity - who checked in, who didn't, plan progress changes, new completions. He has to manually check each client.
#10
No protected video hosting
Training videos are YouTube/Vimeo embeds only. No CDN for protected content. If Marc wants truly private training videos (not just unlisted YouTube links), needs Bunny.net, Mux, or Vimeo OTT integration.
#11
Email templates not unified
Only sendCheckinReplyEmail exists as a reusable template. The invite email builds HTML inline in the API route. Should be a shared template library for consistent branding across all emails.
#12
No lesson reordering in admin
Training lessons can be added and deleted but there's no way to reorder them. order_index exists in the database but no drag-to-reorder or move up/down UI.
#13
Onboarding questionnaire doesn't save
The /onboarding page has a business questionnaire (name, type, turnover, challenges, goals) but there's no API route to persist the answers. The form just transitions to step 3 without saving anything to Supabase.
★
Nice to Have
7 ideas
#14
Client progress overview grid
A single admin page showing all clients in a grid: plan completion %, check-in streak, training progress, last login, status. Right now Marc clicks into each client individually. One glance view would save time.
#15
Check-in reminders via email
The cron route sends push notification reminders, but most clients won't have push enabled on day one. Email fallback for check-in reminders would catch more people and drive engagement.
#16
Direct messaging between Marc and clients
Currently Marc can reply to check-ins, but there's no way for a client to message Marc directly from the portal. A simple message thread per client would replace the need for WhatsApp/email side conversations.
#17
Training progress on client dashboard
Clients can see modules and mark lessons complete, but their dashboard doesn't show overall training completion. A "68% of training complete" progress bar on the homepage would drive engagement.
#18
Export to CSV / PDF
Marc can't export client data, check-in history, or plan progress. Useful for quarterly reviews, accountability calls, or if a client moves on. Simple CSV export per client would cover it.
#19
Light mode option
Currently dark-only. Some construction business owners might prefer light mode, especially on tablets on site. A toggle in settings would be a polish touch.
#20
Mobile bottom navigation bar
The sidebar works on desktop but on mobile it's a hamburger menu. A fixed bottom nav bar (Dashboard, Training, Plan, Check-in, AI) would feel more like a native app - especially since we're pushing PWA install.
